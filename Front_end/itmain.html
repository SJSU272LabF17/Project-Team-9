Checklist for Software Development

1. Project Planning

This is the first step in the software development process. The objective of this phase is to do the following:

a. Develop a Project Definition Document

The project definition document should include the following:

1. Identify stakeholders and their responsibilities

2. Assign project manager

>

3. Provide tools and procedures for requirements management, project tracking, quality management, change management, risk management and release management

b. Develop a project management plan

The project management plan should include the following:

1. Define project schedule

2. Identify resource requirements and allocation

3. Include cost estimation and project budget plan

4. Schedule milestones and reviews

2. Requirement Gathering and Analysis

In this phase, the system inputs, processes, outputs and interfaces are defined in more detail. At the end of this phase, the system requirements will be defined at the functional level.Requirement analysis does not include the programs, files and data streams.

The following steps should be performed in this phase:

1. The Business Analyst develops a plan to carry out requirement gathering and analysis.

2. The Business Analyst must work carefully with the Subject Matter Experts(SME) to generate a Functional Requirements Document.

The Functional Requirements Document can include:

a. Entity-Relationship Diagrams

b. Activity Diagrams

c. Business Algorithms

d. Mock-ups

e. Requirements Traceability Matrix

This phase ends when customer signs off the Functional Requirements Document.

3. Design

The objective of this phase is to transform the identified functional requirements into a detailed system architecture that is feasible, valuable and robust.

The System Design Document is the outcome of the design phase of software development process.

The System Design Document includes the following:

a. Business Application Logic

b. User Interface Design

c. Database Design

d. Process Diagrams

e. Screen Layout Diagrams

4. Coding

The main objective of this phase is to transform the system architecture into a fully-functional working system that is consistent with technical and business requirements.

The coding phase includes the following steps:

a. Coding the software according to approved architecture by satisfying all industry standards

b. Perform unit testing

5. Testing

The objective of this phase is to perform system integration to ensure that the system fulfils all requirements with proper integration of all the system components.

The system may also require additional tests like below depending upon the complexity of the requirements:

a. Security Testing

b. Performance Testing

c. Stress Testing

d. Compatibility Testing

e. Regression Testing

It is necessary that all tests are completed as per the test report and the test results are documented.

6. Deployment

The objective of the deployment phase is to install the system in production environment and to bring it into operation.

The system should fulfil the following criteria:

a. Satisfy the business needs and functional requirements

b. Give System Training

c. Adhere to all Service Level Agreements

d. Operates as per the criteria in the User Manual

e. Create an Implementation Report

7. Maintenance and Support Phase

Maintenance and Support Phase includes the following:

a. Post-Implementation Debugging

b. System corrections, additions, changes and upgrades to ensure that the system meets business requirements.

c. Keep system live

d. Provide user support via helpdesk
